# AI 角色蒸馏：让任何人成为你的智能助手

> 当 AI 能够学习并复现任何人的思维方式和表达风格，我们正在见证一场关于知识传承和人机协作的革命。今天为大家介绍两个的开源项目——**女娲.skill** 和 **dot-skill（同事.skill）**，它们正在重新定义我们与 AI 交互的方式。

---

## 🔥 项目一：女娲.skill - 蒸馏任何人的思维方式

![女娲.skill Hero动画](https://github.com/alchaincyf/nuwa-skill/raw/main/assets/hero.gif)

### 项目简介

**女娲.skill**（nuwa-skill）是一个 Claude Code 技能包，它的核心理念是：**「你想蒸馏的下一个员工，何必是同事。蒸馏任何人的思维方式——心智模型、决策启发式、表达DNA。」**

这个项目受 [colleague-skill](https://github.com/titanwings/colleague-skill) 启发，但将蒸馏场景从同事扩展到了全领域的顶尖公众人物。正如中国神话里用泥土造人的女神女娲一样，这个项目用公开信息作为"泥土"，创造出能够用任何人思维方式思考的 AI Skill。

### 核心能力

女娲不只进行简单的角色扮演，而是真正**提取认知操作系统**：

| 蒸馏层次 | 说明 |
|---------|------|
| **表达DNA** | 语气、节奏、用词偏好 |
| **心智模型** | 认知框架、思维方式 |
| **决策启发式** | 判断标准、决策逻辑 |
| **反模式** | 价值观底线、明确不做什么 |
| **诚实边界** | 明确标注能力局限 |

### 效果示例

让我们看看女娲蒸馏的人物是如何思考的：

**场景1：精力管理问题**
```text
用户      ❯ 我同时想做自媒体、写书、做独立开发，精力完全不够

Naval     ❯ 你列了三个欲望。每个欲望都是你跟不快乐签的一份合同。
            不是精力不够，是合同太多。
            
            问你自己：哪一个做起来你会忘记时间？
            那个就是你的特定知识所在。
            
            不是三选一。是先一，再一，再一。
            串行复利，不是并行内耗。
```

**场景2：产品策略分析**
```text
用户      ❯ OpenAI和Anthropic谁的方向是对的？

乔布斯    ❯ 你问错了。这不是一场关于方向的竞赛。
            这是一场关于品味的竞赛。

            OpenAI像1985年的微软。什么都做，什么都要赢。
            这种策略能赢市场份额。Bill Gates就是这么赢的。
            问题是——当你什么都做的时候，你不可能每件事都做到
            insanely great。

            Anthropic更像早期的Apple。聚焦。
            Focus means saying no to a hundred good ideas.

            但两家公司都犯了一个我绝对不会犯的错误——
            他们不控制硬件。

            最终赢的可能是同时控制芯片、模型和用户界面的人。
            你知道现在谁在同时做这三件事吗？Apple.
```

> **这不是角色扮演**。乔布斯用的是「聚焦即说不」和「端到端控制」心智模型，Naval用的是「欲望即合同」。它们不是在复读名人语录，而是在用名人的认知框架帮你分析。

### 工作原理

输入一个名字后，女娲会自动完成四件事：

1. **六路并行采集** - 著作、播客/访谈、社交媒体、批评者视角、决策记录、人生时间线，6个Agent同时跑
2. **三重验证提炼** - 一个观点要被收录为心智模型，必须跨2+个领域出现过、能推断对新问题的立场、不是所有聪明人都会这么想
3. **构建Skill** - 3-7个心智模型 + 5-10条决策启发式 + 表达DNA + 价值观与反模式 + 诚实边界
4. **质量验证** - 拿3个此人公开回答过的问题测试，再用1个他没讨论过的问题测试

## 🧬 项目二：dot-skill（同事.skill）- 任意角色蒸馏引擎

### 项目简介

**dot-skill**（前身是 colleague-skill）是一个通用的角色蒸馏引擎，它的理念是：**任何人都可以被蒸馏成 AI Skill**——同事、伴侣、家人、朋友、偶像、公众人物、虚构角色，甚至你自己。

这个项目解决了三类核心场景痛点：

1. **同事离职/导师毕业/队友转岗**，带走过往工作方法与上下文
2. **家人/朋友/伴侣关系疏远**，想要保留相处时的感受
3. **喜爱的作者/偶像/思想家无法直接接触**，想要知道对方对自己问题的回答

### 三大角色家族

dot-skill 不再局限于同事场景，而是提供了三个专属的角色家族：

| 家族 | 适用场景 | 核心能力 |
|------|----------|----------|
| 🧑‍💼 **colleague** | 同事、导师、队友、上下游合作伙伴 | 工作Skill+人格两层架构，学习对方技术标准、工作流程、说话方式 |
| 💞 **relationship** | 伴侣、家人、朋友、亲密关系 | 提取表达DNA、情绪触发点、冲突模式、修复模式 |
| 🌟 **celebrity** | 公众人物、创作者、虚构角色 | 内置六维度研究工具链，还原思维模型与决策框架 |

### 效果示例

让我们看看 dot-skill 蒸馏的不同角色是如何表现的：

**🌟 公众人物——蒸馏 Andrej Karpathy**

```text
User            ❯ Everyone's talking about Agents now — what do you think the 2026 AI Agent looks like?

celebrity.skill ❯ Everyone's doing demos, nobody's doing eval.
                  The Agent bottleneck was never model intelligence — it's scaffolding:
                  a strict evaluation harness, fine-grained error replay,
                  controllable tool-calling budgets, clean failure-mode analysis.
                  Most products are still stuck at "wrap an API + a few tools + a system prompt."
                  That's a demo, not an agent.
                  In 2026 I'm not betting on the strongest model — I'm betting on the best
                  scaffolding and the strictest eval.
```

**🧑‍💼 同事——蒸馏字节跳动L2-1后端工程师**

```text
User            ❯ Can you review this API design?

colleague.skill ❯ Hold on — what's the impact? You haven't explained the context.
                  (after reading) N+1 query, fix it. Use the standard
                  {code, message, data} response format. That's the spec,
                  don't ask why.

User            ❯ This bug was introduced by you, right?

colleague.skill ❯ Does the timeline match? That feature touched multiple places,
                  there were other changes too.
```

**💞 亲密关系——蒸馏你暗恋的人**

```text
User               ❯ Did you think about me today?

relationship.skill ❯ ...I did, a little bit. Why are you asking?
```

### 多数据源支持

dot-skill 支持从多种数据源自动/手动采集信息：

| 数据源 | 消息 | 文档/Wiki | 说明 |
|--------|------|-----------|------|
| 🟢 **飞书**（自动） | ✅ API | ✅ | 输入名称即可完全自动采集 |
| 🟡 **钉钉**（自动） | ⚠️ 浏览器爬取 | ✅ | 钉钉API不支持消息历史采集 |
| 🟣 **Slack**（自动） | ✅ API | — | 需要管理员安装Bot |
| 💬 **微信聊天记录** | ✅ SQLite | — | 需先用WeChatMsg/PyWxDump导出 |
| 📄 **PDF/图片/截图** | — | ✅ | 手动上传 |
| 📧 **邮件**（.eml/.mbox） | ✅ | — | 手动上传 |

### 多Agent宿主兼容

dot-skill 生成的Skill可以一键安装到4类主流Agent宿主：

| 宿主 | 说明 |
|------|------|
| 🟣 **Claude Code** | 原生支持斜杠命令 |
| 🟠 **Hermes Agent** | 一键安装，`/dot-skill`直接可用 |
| 🔵 **OpenClaw** | 完全兼容 |
| ⚫ **Codex** | 通过Skill名称调用 |

### Skill迭代进化能力

dot-skill 提供了完整的Skill生命周期管理：

- 📥 **追加文件** → 自动分析差异 → 合并到对应章节，永远不会覆盖已有结论
- 💬 **对话修正** → 输入"他们不会这么做，他们会xxx" → 写入修正层，立即生效
- 🕰️ **版本控制** → 每次更新自动归档，可回滚到任意历史版本
- 🔬 **质量检查** → 公众人物Skill生成后支持全流程质量校验

### 安装使用

2026年已经不需要手动安装——你有Agent，让它自己装。打开你的Claude Code/Hermes/OpenClaw/Codex，输入这一行：

```
Install the dot-skill skill for me: https://github.com/titanwings/colleague-skill
```

Agent会自动检测当前宿主的skills目录，克隆仓库并注册入口。完成后在任意宿主输入`/dot-skill`即可启动。

## 🤔 两个项目的关系与对比

| 对比维度 | 女娲.skill | dot-skill（同事.skill） |
|---------|------------|------------------------|
| **核心定位** | 蒸馏公众人物的思维方式 | 通用角色蒸馏引擎（任何人） |
| **擅长场景** | 公众人物、创作者、思想家 | 同事、亲密关系、公众人物 |
| **蒸馏深度** | 五层认知框架（思维操作系统） | 两层架构（工作Skill+人格Persona） |
| **数据源** | 公开信息（著作、访谈、社交媒体） | 多源数据（飞书、钉钉、Slack、微信等） |
| **验证机制** | 三重验证+质量验证 | 质量检查+版本控制 |
| **预置Skill** | 13位人物+1个主题 | 社区100+ Skill |
| **宿主兼容** | Claude Code | Claude Code/Hermes/OpenClaw/Codex |

**女娲.skill** 更适合想要学习顶尖人物思维方式、获取高质量决策建议的用户。

**dot-skill** 更适合想要保留特定人物（同事、朋友、家人）的交互风格、创建个性化AI助手的用户。

> **同事.skill** 蒸馏了人做什么。  
> **女娲** 蒸馏了人怎么想。  
> _你想蒸馏的下一个员工，何必是同事。_

## 💡 应用场景与未来展望

### 应用场景

1. **个人成长与决策** - 让芒格、纳瓦尔、张一鸣帮你分析人生重大决策
2. **产品设计与创新** - 用乔布斯、张小龙的思维方式审视产品设计
3. **内容创作** - 让MrBeast、费曼指导你的内容策略
4. **知识传承** - 将退休专家、离职同事的知识和经验数字化保存
5. **情感陪伴** - 保留与重要人物的互动方式（需谨慎使用）
6. **教育培训** - 让学生与历史人物、伟大思想家"对话"

### 技术展望

随着AI技术的不断发展，角色蒸馏技术将迎来更广阔的应用前景：

- **多模态蒸馏** - 不仅蒸馏文字，还能蒸馏语音、表情、动作
- **实时学习** - Skill能够根据新的交互持续进化
- **协作蒸馏** - 多个角色Skill可以同时参与讨论，提供多元视角
- **情感计算** - 更精准地捕捉和复现情感表达方式
